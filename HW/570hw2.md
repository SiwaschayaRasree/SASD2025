Code smells คือ คือหลักการที่เอาไว้ตรวจว่า โค้ดที่เราออกแบบไว้มันไปผิดหลักการออกแบบหรือเปล่า 
ซึ่งถ้าผิดมันหมายความว่าโครงสร้างพวกนั้นจะมีปัญหาในอนาคต และมีอีกชื่อนึงคือ Design Smells

a) ความหมายของ Incomplete Library Class
Incomplete Library Class คือ  "กลิ่นแปลกๆ ในโค้ด" (Code Smell) ที่เกิดขึ้นเมื่อคลาสในไลบรารีที่นำมาใช้งาน ไม่มีความสามารถเพียงพอต่อการตอบสนองความต้องการของระบบ 
ทำให้นักพัฒนาต้องสร้างคลาสใหม่หรือใช้วิธีการเสริม (เช่น Wrapper, Inheritance) เพื่อเพิ่มเติมฟังก์ชันการทำงานที่ขาดหายไป 

b) ตัวอย่างโค้ดที่เกิด Incomplete Library Class
// Library class (แก้ไขไม่ได้)
public class Point {
    public int x;
    public int y;

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}


c) ตัวอย่างโค้ดที่ refactoring แล้ว โดยใช้ wrapper class
public class ExtendedPoint {
    private Point point;

    public ExtendedPoint(Point point) {
        this.point = point;
    }

    public double distanceTo(ExtendedPoint other) {
        return Math.sqrt(
            Math.pow(point.x - other.point.x, 2) +
            Math.pow(point.y - other.point.y, 2)
        );
    }
}


ExtendedPoint p1 = new ExtendedPoint(new Point(1, 2));
ExtendedPoint p2 = new ExtendedPoint(new Point(4, 6));

double d = p1.distanceTo(p2);


ไม่จำเป็นต้อง refactor ถ้าใช้ครั้งเดียวหรือ refactor แล้วไม่คุ้ม
